# 資通安全職能訓練完整指引

根據資通安全職能訓練教材（11501 v4）整理之完整技術指引。

---

## 一、資通安全核心概念

### 1.1 資安三要素 (CIA Triad)

| 要素 | 英文 | 定義 | 範例 |
|-----|------|------|------|
| 機密性 | Confidentiality | 確保資訊不被未授權者存取 | 加密、存取控制 |
| 完整性 | Integrity | 確保資訊未被未授權修改 | 雜湊、數位簽章 |
| 可用性 | Availability | 確保授權使用者可存取資訊 | 備份、備援 |

### 1.2 資訊安全延伸特性

- **鑑別性 (Authentication)**：確認使用者身分
- **授權性 (Authorization)**：確認使用者權限
- **可歸責性 (Accountability)**：追蹤使用者行為
- **不可否認性 (Non-repudiation)**：無法否認已執行之行為

## 二、資通系統防護控制措施（7構面29項）

### 2.1 存取控制構面

| 措施 | 高 | 中 | 普 | 說明 |
|-----|---|---|---|------|
| 帳號管理 | ✓ | ✓ | ✓ | 唯一帳號、禁止共用、定期盤查 |
| 最小權限 | ✓ | ✓ | ✓ | 僅授權業務所需最小權限 |
| 遠端存取 | ✓ | ✓ | ✓ | 加密連線、來源管制、連線監控 |
| 存取授權 | ✓ | ✓ | ✓ | Need-to-Know、職務區隔 |

**授權原則**：
- **業務僅知 (Need-to-Know)**：僅提供執行業務所需資訊
- **最小權限 (Least Privilege)**：僅授權最低操作權限
- **職務區隔 (Separation of Duties)**：避免利益衝突職務由同一人執行
- **特權管理 (PAM)**：管理者帳號須特別控管並記錄

### 2.2 事件日誌與可歸責性構面

| 措施 | 高 | 中 | 普 | 說明 |
|-----|---|---|---|------|
| 記錄事件 | ✓ | ✓ | ✓ | 日誌保留至少6個月 |
| 日誌內容 | ✓ | ✓ | ✓ | 事件類型、時間、位置、使用者ID |
| 日誌容量 | ✓ | ✓ | ✓ | 依需求配置儲存空間 |
| 失效回應 | ✓ | ✓ | ✓ | 日誌處理失效時即時通報 |
| 時戳校時 | ✓ | ✓ | ✓ | 使用UTC/GMT標準時間 |
| 日誌保護 | ✓ | ✓ | ✓ | 存取控制、完整性驗證、定期備份 |

**日誌類型**：
- OS event log（作業系統日誌）
- Web log（網站日誌）
- AP log（應用程式日誌）
- Logon log（登入日誌）

### 2.3 營運持續計畫構面

| 措施 | 高 | 中 | 普 | 說明 |
|-----|---|---|---|------|
| 系統備份 | ✓ | ✓ | ✓ | RPO設定、定期測試、異地備份 |
| 系統備援 | ✓ | ✓ | - | RTO設定、備援設備 |

**關鍵時間指標**：

```
最近備份 ─────── 災害發生 ─────── 系統恢復 ─────── 業務復原
    ←── RPO ──→ ←────── RTO ──────→ ←── WRT ──→
    └────────────────── MTPD ─────────────────────┘
```

- **RPO (Recovery Point Objective)**：可容忍資料損失時間
- **RTO (Recovery Time Objective)**：可容忍服務中斷時間
- **WRT (Work Recovery Time)**：工作復原時間
- **MTPD (Maximum Tolerable Period of Disruption)**：最大可容忍中斷時間

**備份方式**：
| 方式 | 說明 | 優點 | 缺點 |
|-----|------|------|------|
| 完整備份 | 完整複製所有檔案 | 恢復最完整 | 耗時長、空間大 |
| 差異備份 | 備份上次完整備份後變更檔案 | 較快、空間較少 | 還原需完整+差異 |
| 增量備份 | 備份上次備份後變更檔案 | 最快、空間最少 | 還原最複雜 |

**備份原則**：
- 3-2-1原則：3份備份、2種媒體、1份異地
- 異地備援距離 ≥ 30公里

### 2.4 識別與鑑別構面

| 措施 | 高 | 中 | 普 | 說明 |
|-----|---|---|---|------|
| 內部使用者識別 | ✓ | ✓ | ✓ | 唯一識別、禁止共用帳號 |
| 身分驗證管理 | ✓ | ✓ | ✓ | 密碼複雜度、帳戶鎖定、OTP |
| 鑑別資訊回饋 | ✓ | ✓ | ✓ | 遮蔽鑑別資訊 |
| 加密模組鑑別 | ✓ | ✓ | - | 密碼加密或雜湊儲存 |
| 多因子認證 | ✓ | - | - | MFA強固鑑別 |

**身分鑑別三因素**：
- **所知 (Something You Know)**：密碼、PIN碼
- **所有 (Something You Have)**：晶片卡、OTP Token、手機
- **與生俱備 (Something You Are)**：指紋、虹膜、臉部辨識

**OTP技術**：
- **同步式OTP**：基於時間或事件計數同步
- **非同步式OTP**：基於伺服器發出的挑戰值

**密碼管理要求**：
- 長度至少8碼
- 包含文字、數字、符號、大小寫
- 登入失敗5次鎖定至少15分鐘
- 不可與前3次密碼相同
- 定期更換、禁止明文傳輸與儲存

### 2.5 系統與服務獲得構面（SSDLC）

| 階段 | 高 | 中 | 普 | 安全要求 |
|-----|---|---|---|----------|
| 需求階段 | ✓ | ✓ | ✓ | 確認安全需求（CIA） |
| 設計階段 | ✓ | ✓ | - | 威脅識別、風險評估 |
| 開發階段 | ✓ | ✓ | ✓ | 安全編碼、源碼掃描 |
| 測試階段 | ✓ | ✓ | ✓ | 弱點掃描、滲透測試 |
| 部署維運 | ✓ | ✓ | ✓ | 版本控制、變更管理 |
| 委外階段 | ✓ | ✓ | ✓ | 契約納入安全需求 |

**安全檢測類型**：
| 檢測類型 | 適用等級 | 說明 |
|---------|---------|------|
| 弱點掃描 | 高中普 | 自動化工具掃描已知弱點 |
| 源碼掃描 | 高 | 分析源碼安全弱點 |
| 滲透測試 | 高 | 模擬攻擊驗證安全性 |

### 2.6 系統與通訊保護構面

| 措施 | 高 | 中 | 普 | 說明 |
|-----|---|---|---|------|
| 傳輸加密 | ✓ | - | - | TLS 1.2+、SSH、SFTP、VPN |
| 資料儲存 | ✓ | - | - | 組態檔及機敏資訊加密儲存 |

**加密演算法**：

| 類型 | 演算法 | 說明 |
|-----|--------|------|
| 對稱式 | AES-256 | 加解密同一金鑰，速度快 |
| 非對稱式 | RSA、ElGamal | 公私鑰配對，金鑰管理容易 |
| 雜湊 | SHA-256、SHA-512 | 固定長度摘要，不可逆 |

**數位信封**：結合對稱式（加密資料）與非對稱式（加密金鑰）優點

**數位簽章**：提供完整性、鑑別性、不可否認性
- 發送方用私鑰簽署訊息摘要
- 接收方用公鑰驗證簽章

### 2.7 系統與資訊完整性構面

| 措施 | 高 | 中 | 普 | 說明 |
|-----|---|---|---|------|
| 漏洞修復 | ✓ | ✓ | ✓ | 定期更新、測試有效性 |
| 系統監控 | ✓ | ✓ | ✓ | 偵測攻擊、入侵通報 |
| 軟體完整性 | ✓ | ✓ | - | 完整性驗證、伺服器端檢查 |

## 三、技術安全防護措施

### 3.1 防毒軟體

**惡意程式類型**：
| 類型 | 說明 |
|-----|------|
| 病毒 (Virus) | 感染合法程式進行傳播 |
| 蠕蟲 (Worm) | 自我複製、網路傳播 |
| 間諜軟體 (Spyware) | 監控使用者、竊取資訊 |
| 後門程式 (Backdoor) | 繞過認證、遠端控制 |
| 木馬程式 (Trojan) | 偽裝合法軟體執行惡意行為 |

**部署架構**：
```
Internet → 防火牆 → DMZ (SMTP防毒) → 內部網段 (HTTP/FTP防毒) → 端點防毒
```

**管理重點**：
- 定期自動更新病毒碼
- 配合NAC控管未安裝防毒設備
- 個人端與閘道端採用不同廠牌（縱深防禦）

### 3.2 防火牆

**類型**：
| 類型 | 運作層級 | 特點 |
|-----|---------|------|
| 封包過濾 | L3-L4 | 檢查IP/Port，速度快 |
| 狀態檢測 | L3-L4 | 追蹤連線狀態 |
| 應用層閘道 | L7 | 深度封包檢測 |
| 次世代防火牆 (NGFW) | L3-L7 | 整合IPS、應用識別、URL過濾 |

**部署原則**：
- 預設拒絕所有連線
- 僅開放業務必要連接埠
- 定期審查規則、移除冗餘

### 3.3 Web應用程式防火牆 (WAF)

**防護範圍**：OWASP Top 10
- SQL Injection
- XSS (Cross-Site Scripting)
- CSRF (Cross-Site Request Forgery)

**偵測技術**：
- 負面模型：阻擋已知攻擊特徵
- 正面模型：僅允許符合規則的請求

### 3.4 入侵偵測/防禦系統 (IDS/IPS)

| 項目 | IDS | IPS |
|-----|-----|-----|
| 功能 | 偵測並警示 | 偵測並阻擋 |
| 部署 | 旁聽模式 (TAP) | 串接模式 (Inline) |
| 誤判影響 | 僅誤報 | 可能阻斷正常流量 |

**類型**：
- **NIDS/NIPS**：網路型，監控網路流量
- **HIDS/HIPS**：主機型，監控主機活動

**偵測方法**：
- 簽章比對 (Signature-based)
- 異常偵測 (Anomaly-based)
- 行為分析 (Behavioral)

### 3.5 SOC 資安監控中心

**核心功能**：
- 7×24 即時監控
- 資安事件分析與調查
- 威脅獵捕 (Threat Hunting)
- 情資整合與分享

**成熟度等級**：
| 等級 | 能力 |
|-----|------|
| Level 1 | 被動反應 |
| Level 2 | 主動監控 |
| Level 3 | 威脅獵捕 |
| Level 4 | 情資驅動 |
| Level 5 | 自動化應變 |

**關鍵工具**：
- SIEM：安全資訊與事件管理
- SOAR：安全協調、自動化與回應
- EDR：端點偵測與回應
- NDR：網路偵測與回應
- TIP：威脅情資平台

### 3.6 EDR 端點偵測與回應

**功能**：
- 端點威脅偵測
- 即時應變處置
- 威脅獵捕
- 行為分析

**與防毒軟體差異**：
| 項目 | 防毒軟體 | EDR |
|-----|---------|-----|
| 偵測方式 | 簽章比對 | 行為分析 |
| 反應能力 | 隔離/刪除 | 調查/回應/獵捕 |
| 可見度 | 檔案層級 | 行為/程序層級 |

## 四、媒體控管與可攜式設備

### 4.1 媒體控管工作

1. **標示**：名稱、版本、機密等級、建立日期、保存期限
2. **使用紀錄**：記錄使用情況
3. **完整性檢查**：定期檢查資料完整性
4. **實體存取保護**：限制未授權存取
5. **運送保護**：確保運送安全
6. **重用安全**：資料安全清除
7. **銷毀丟棄**：實體破壞

### 4.2 安全資料刪除

| 媒體類型 | 清除方式 |
|---------|---------|
| HDD | 單次覆寫 (Single Overwrite) |
| SSD | 加密式清除 (Cryptographic Erase) |
| 手機 | 原廠重設 + 金鑰刪除 |

### 4.3 媒體銷毀

| 媒體 | 銷毀方式 |
|-----|---------|
| 磁碟機 | 消磁或碎裂(≤6mm) |
| 磁帶 | 消磁 |
| 光碟 | 碎裂(≤6mm) |

## 五、進階持續性威脅 (APT)

### 5.1 APT 特徵

- **針對性**：鎖定特定組織或產業
- **持續性**：長期潛伏、持續滲透
- **隱匿性**：規避傳統偵測機制
- **多階段**：情蒐 → 入侵 → 橫向移動 → 資料竊取

### 5.2 APT 攻擊階段

1. **偵察**：蒐集目標資訊
2. **武器化**：製作惡意程式
3. **傳遞**：釣魚郵件、惡意網站
4. **利用**：觸發漏洞
5. **安裝**：植入後門
6. **命令與控制**：建立 C2 通道
7. **行動達成**：竊取資料

### 5.3 APT 防禦策略

- 多層次縱深防禦
- 異常行為偵測
- 威脅情資整合
- 使用者資安意識訓練
- 網路分段與微分段
- 零信任架構

## 六、雜湊函式與密碼安全

### 6.1 雜湊特性

- **固定長度**：任意長度輸入產生固定長度輸出
- **單向性**：無法從雜湊值回推原始資料
- **抗碰撞性**：難以找到相同雜湊值的不同輸入

### 6.2 常用演算法

| 演算法 | 輸出長度 | 安全性 |
|--------|---------|--------|
| MD5 | 128 bits | 已不安全 |
| SHA-256 | 256 bits | 安全 |
| SHA-512 | 512 bits | 安全 |
| SHA3-512 | 512 bits | 安全 |

### 6.3 Salt 加鹽技術

**用途**：防禦彩虹表攻擊與碰撞攻擊

**原理**：
```
最終雜湊值 = Hash(密碼 + Salt)
```

**特點**：
- 每個使用者使用唯一隨機 Salt
- 即使密碼相同，雜湊值也不同

## 七、符記 (Token) 類型

| 類型 | 特點 |
|-----|------|
| 記憶卡 | 僅儲存資訊，無運算能力 |
| 智慧卡 | 有微處理器，可運算與儲存 |

**智慧卡特質**：
- 具備微處理器與積體電路
- 可記憶也可運算
- 防拆解與竄改保護
- 須PIN碼啟動
