# 台灣資通安全法規與實務知識庫

整合數位發展部資通安全署（moda/ACS）、國家資通安全研究院（NICS）與資安職能訓練教材的完整資安法規、技術控制與實務指引。

## 適用情境

使用此 skill 當你需要：
- 了解台灣《資通安全管理法》及其6項子法
- 查詢資安責任等級分級要求（A/B/C/D/E級）
- 了解系統防護基準（7構面29項控制措施）
- 查閱 NIST CSF 2.0 框架與 NICS 共通規範
- 了解 GCB、VANS、ZTA、N-SOC、EDR 等防護機制
- 確認資安專業證照或訓練證書清單與時數要求
- 了解資安事件通報應變流程與時限
- 查閱政府機關資安合規與委外指引
- 了解大陸廠牌資通訊產品限制規定
- 理解風險管理、BCP營運持續計畫（RPO/RTO/MTPD）
- 查詢加密技術、數位簽章、身分鑑別（MFA/OTP）規範
- 了解 SSDLC 安全系統發展生命週期要求

## 一、核心法規架構

### 資通安全管理法

**最新版本**：114年9月24日總統公告修正版

**主要內容**：
- 規範公務機關及特定非公務機關的資安管理責任
- 建立資安責任等級分級制度（A/B/C/D/E 五級）
- 規定資安事件通報及應變機制
- 授權訂定相關子法

### 子法清單

| 子法名稱 | 主要內容 |
|---------|---------|
| 資通安全管理法施行細則 | 定義、維護計畫、委外規範 |
| 資通安全責任等級分級辦法 | 機關責任等級 A/B/C/D/E 分級標準 |
| 資通安全事件通報及應變辦法 | 資安事件通報時限與應變程序 |
| 資通安全情資分享辦法 | 情資分享機制與平台 |
| 危害國家資通安全產品審查辦法 | 限制使用產品之審查程序 |
| 資通安全維護計畫實施情形稽核辦法 | 稽核程序與要求 |

## 二、資安責任等級

### 分級標準

依據《資通安全責任等級分級辦法》：

| 等級 | 適用對象 | 專職人員 | 主要要求 |
|-----|---------|---------|---------|
| A 級 | 國防、外交等核心機關 | ≥4名 | ISMS驗證、GCB、VANS、SOC、滲透測試 |
| B 級 | 中央二級機關及直轄市 | ≥2名 | ISMS驗證、GCB、VANS、資安健診 |
| C 級 | 中央三級機關及縣市 | ≥1名 | 資安維護計畫、資安健診 |
| D 級 | 鄉鎮市區公所等 | - | 基本資安防護 |
| E 級 | 其他機關 | - | 最低資安要求 |

### 資通系統防護需求分級

依附表九「資通系統防護需求分級原則」：
- **高**：支持核心業務、涉及機敏資料
- **中**：重要業務系統
- **普**：一般業務系統

### 系統防護基準（7構面29項控制措施）

依《資通安全責任等級分級辦法》附表十，資通系統應依防護需求等級實施：

| 構面 | 控制項目 | 高 | 中 | 普 |
|-----|---------|---|---|---|
| 存取控制 | 帳號管理、最小權限、遠端存取 | ✓ | ✓ | ✓ |
| 事件日誌 | 記錄事件、日誌保護、時戳校時 | ✓ | ✓ | ✓ |
| 營運持續 | 系統備份、系統備援 | ✓ | ✓ | 部分 |
| 識別鑑別 | 身分驗證、MFA、加密模組 | ✓ | 部分 | 部分 |
| 系統獲得 | SSDLC各階段安全需求 | ✓ | ✓ | ✓ |
| 通訊保護 | 傳輸加密、資料儲存安全 | ✓ | - | - |
| 資訊完整 | 漏洞修復、系統監控、完整性驗證 | ✓ | ✓ | 部分 |

**詳細控制措施說明**：請參閱 `references/training_guide.md`

## 四、NICS 核心防護機制

### GCB - 政府組態基準

**定義**：政府機關資通訊設備的安全組態標準

**適用對象**：A級、B級機關（初次核定或等級變更後1年內完成）

**公告項目**：
- 作業系統（Windows、Linux）
- 瀏覽器（Chrome、Edge、Firefox）
- 網通設備
- 應用程式

**導入原則**：
- 專版專用
- 導入前充分測試
- 例外管理須審核與定期檢討

### VANS - 弱點通報機制

**定義**：資訊資產弱點管理與通報系統

**導入範圍**：
- 公務機關：全機關資訊資產
- 關鍵基礎設施：關鍵資訊基礎設施及營運必要系統

**管理要求**：
- 每季至少上傳資產盤點資料
- CVSS 7.0分以上弱點優先修補
- 未能即時修補須規劃緩解措施
- 每季比對弱點後1個月內填寫處置方式

### ZTA - 零信任架構

**核心原則**：永不信任，持續驗證（Never Trust, Always Verify）

**實施重點**：
- 身分鑑別強化
- 設備健康檢查
- 最小權限存取
- 持續監控與驗證

### N-SOC - 國家資安聯防監控中心

**功能**：
- 全國資安威脅監控
- 資安事件協處
- 情資分享與預警

### EDR - 端點偵測應變

**功能**：
- 端點威脅偵測
- 即時應變處置
- 威脅獵捕
- 行為分析

### 技術安全防護設備

| 設備類型 | 功能 | 部署位置 |
|---------|------|---------|
| 防火牆 | 封包過濾、狀態檢測 | 網路邊界 |
| NGFW | 整合IPS、應用識別 | 網路邊界 |
| WAF | Web應用程式防護 | DMZ |
| IDS/IPS | 入侵偵測/防禦 | 網路/主機 |
| SIEM | 日誌收集與分析 | SOC |
| SOAR | 自動化應變 | SOC |
| EDR | 端點偵測應變 | 端點 |
| NDR | 網路偵測應變 | 網路 |

### 安全檢測類型

| 檢測類型 | 適用等級 | 說明 |
|---------|---------|------|
| 弱點掃描 | 高中普 | 自動化工具掃描已知弱點 |
| 源碼掃描 | 高 | 分析源碼安全弱點 |
| 滲透測試 | 高 | 模擬攻擊驗證安全性 |
| 資安健診 | B級以上 | 綜合性安全評估 |

## 五、證照與訓練

### 資安專業證照清單

依《資通安全責任等級分級辦法》第11條，A/B/C級機關專職人員須持有證照：

**國際認證**：
- CISSP（資訊系統安全專業認證）
- CISM（資訊安全管理師）
- CEH（道德駭客認證）
- CISA（資訊系統稽核師）
- OSCP（滲透測試專家）
- CompTIA Security+

**國內認證**：
- iPAS 資訊安全工程師
- 資安技術專業人員

### 資安職能訓練證書

**取得方式**：
- 參加主管機關認證之訓練機構課程
- 完成指定時數並通過評量

**時數要求**：
- A級機關專職人員：每年12小時專業課程 + 3小時通識
- B級機關專職人員：每年12小時專業課程 + 3小時通識
- C級機關專職人員：每年12小時專業課程 + 3小時通識

## 七、重要作業指引

### 政府機關雲端服務應用資安要求

1. **禁止大陸雲端服務**：不得使用大陸地區（含港澳）廠商之雲端服務
2. **資通訊產品限制**：不得使用大陸廠牌產品（含軟硬體及服務）
3. **人員限制**：境內執行團隊不得有陸籍人士
4. **資料存放**：不得存放於大陸地區，不得跨境傳輸
5. **白牌設備**：雲端服務提供者自行設計之白牌設備暫不納入限制

### 大陸廠牌限制規定

**依據**：行政院109年12月18日院臺護字第1090201804A號函

**限制範圍**：
- 大陸地區廠商所提供之資通訊產品
- 陸籍人士參與執行
- 目前僅限制最終產品，暫未限制零組件

**注意事項**：
- 香港、澳門居民不在陸籍限制範圍
- 多重國籍含大陸地區人民亦受限制
- 各機關可依個案評估限制零組件

### 委外資安要求

依《資安法施行細則》第4條：
- 受託者應具備完善資安管理措施或通過第三方驗證
- 客製化系統開發須提供安全性檢測證明（源碼掃描、弱點掃描、滲透測試）
- 委託金額達1,000萬以上之客製化核心系統，須由機關自行或另委託第三方辦理安全性檢測

## 八、常見問題 (FAQ)

### Q1: 資通安全與資訊安全有何不同？
資通安全涵蓋資訊與通信，範圍較資訊安全廣泛，目前多以「資通安全」稱之。

### Q2: 如何進行情資分享？
依《資通安全情資分享辦法》第3條第3項，可於國家資通安全通報應變網站以一般機關身分登入，選擇「情資分享功能」填報。

### Q3: 如何評估資通系統防護需求？
應依附表九「資通系統防護需求分級原則」，就機關業務屬性、系統特性及資料持有情形，訂定客觀量化指標進行評估。

### Q4: 資安專職人員與專責人員有何不同？
- **專職人員**（公務機關）：全職執行資安業務
- **專責人員**（特定非公務機關）：有專人負責資安事務，無全職要求

### Q5: 日誌應保留多久？
各機關應保留日誌至少6個月，建議定期備份至與原日誌系統不同之實體。

### Q6: 異地備份距離要求？
參考「我國電腦機房異地備援機制參考指引」，主機房與異地備援機房距離應30公里以上。
